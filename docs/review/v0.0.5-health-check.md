# biliHoyoFairy v0.0.5 — 深度体检报告（多 Agent 评审汇总）

> 评审对象：`docs/review/v0.0.5-snapshot.user.js`（单文件单 IIFE，纯 JS，3383 行，main 分支 v0.0.5）。
> 方法：5 个只读 Agent 并行体检（架构 / 正确性 / 安全 / 性能 / UX），本文为汇总与定级。
> 行号均指 **snapshot**。严重度：S0 致命 · S1 高 · S2 中 · S3 轻。

---

## 0. 总评

v0.0.5 是一份**防御意识不错、设计有亮点**的单文件脚本：HTML 注入处普遍走 `escapeHtml`、原型链污染在所有合并路径都有防护、CSRF token 只发往写死的 `api.bilibili.com`、账号写操作都在显式点击后触发。没有发现 XSS、`eval`/远程执行、凭据外泄。

但它距离"专业工具"有明确差距，集中在四块：
1. **架构**：全部挤在一个 IIFE、零导出、纯逻辑与 `CONFIG`/DOM/GM 副作用纠缠 → 不可测、不可复用；拆分时存在 `dom↔rules` 真实环。
2. **正确性/性能**：批量拉黑每条都触发"全量重建规则 + 全页重扫 + 存盘"（O(N·卡片)）；feed 过滤无逐项容错（一条异常废掉整个响应）。
3. **安全**：`@connect *` 过宽（SSRF/信标）；导入配置不剥离 `subscriptions`（可被植入自动联网 URL）；订阅里的 `/正则/` 可 ReDoS 卡死页面。
4. **UX/安全**：单击拉黑**零确认**且紧挨本地屏蔽；批量不可中断、"当前页批量"无进度；无引导、无深色、可达性弱。

> 注：v0.0.5 单文件**没有**"停止"按钮，单击拉黑**没有**二次确认（这是体检结论，不是我之前模块分支的状态）。

各分项详报可追溯到子 Agent：[架构](a3709568-1d50-48be-92ee-63a61ed871f1) · [正确性](c7ef70f2-837c-4b0d-8e86-1a615b9762bf) · [安全](f26d9bf7-e771-4ec3-89f7-1cb951f13ecc) · [性能](27d5f7ed-63fc-47f8-9770-fd606cc72803) · [UX](70b8d9e3-4b71-4a25-ad76-b92a1265fff7)。

## 1. 必须保留的"三件套"（设计亮点）

1. **维度注册表** `SYNC_DIMS`/`API_DIMS`（691–763）：schema 化声明，`matchRule`/`matchApi`/`apiNeeds` 自动派生。加维度 = 加一条。**这是全仓最佳设计。**
2. **可插拔网络管线** `NET`（pre/post，1024–1061）+ `FEED_HOOKS` 端点表（977–995）：拦截层的核心。
3. **`M`/`ruleVersion` live-binding 重建模型**（634/667/668）：规则变更后全模块读到最新匹配器。

re-mod 只改"打包方式"，不改这三者的形状。

### S3 — 可选/打磨

- 正则规则绕过 fuzzy/全角/小写归一（`textHit` 1441 起；evasion gap，正则预置库受影响）。
- 非法用户正则被静默吞掉、chip 仍显示"已生效"（422–432）。
- `parseExpires` 把未知单位（如 `30m`）当天处理（249–254）。
- `boostFeedLoad` 把 `ps` 写死 30，可能反而调小（1056–1061）→ `Math.max(orig,30)`。
- 风控触发的那一条被记为 `failed` 不重排（页批量/联合投稿不回填）。
- 评论占位符用 `replace(/[<>&]/g,'')` 而非统一 `escapeHtml`（1478–1481，安全但不统一）。
- 源徽章 黑/拦 同绿色、无图例（3203–3209）；emoji/符号风格不一致（✅ vs ✗）。
- in-session API 缓存（view/tag/card Map，848）SPA 下无界增长。
- 文案多处口语（"拿不到""粘个来试""稍慢"），见 UX 报告 before→after 表。

## 3. 目标模块结构（架构 Agent 提案，re-mod 蓝图）

严格分层 DAG、无上行依赖、无环。相对现 `src/` 唯一结构性改动：引入 `events`(`onRulesChanged`) seam 断 `dom↔rules` 环。

```
L0 纯叶（可单测、无全局）   constants · util · match/normalize(fuzzy 注入)
L1 状态+数据              config(CONFIG/load/save/import/export) · logging · presets · subscriptions/parse
L2 领域逻辑              subscriptions/store · cardinfo(flags 注入) · match/engine(M,ruleVersion,SYNC/API_DIMS,matchRule/matchApi)
L3 IO/副作用            api(riskGuard+queue) · stats(纯计数+监听) · net(FEED_HOOKS+NET) · dom/shadow · events(onRulesChanged)
L4 编排/DOM             page · dom(scan/block/rescan) · comments · rules(改CONFIG→发事件) · blacklist/batch(parseList/addLocalMany 纯函数) · blacklist(relation/modify) · hotsearch · subscriptions/refresh(发事件)
L5 UI                  ui/hooks(注入) · ui/toast(badge+toast) · ui/field(通用列表字段组件) · ui/panel · ui/menu
顶层                   main(bootstrap+observer+wiring+installShadowHook)
```

**与现 `src/` 对比**：`engine/normalize/net/api/subscriptions/*` 边界对、应保留；要**重切**的是：① 断 `dom↔rules` 环（加 events）；② 拆解 1240 行的 `ui/panel` god-module（抽 `ui/field` + DOM-free 的 `blacklist/batch`，订阅 URL 载入改走 `subscriptions/refresh`）；③ `cardinfo` 去 `CONFIG` 耦合；④ **全层去 `@ts-nocheck`** 让类型检查器守住分层。
